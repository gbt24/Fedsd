# De-fingerprint Experiment

This experiment investigates fingerprint removal methods for federated learning diffusion models by evaluating fine-tuning as a defingerprinting attack.

## Quick Start

### 1. Run Experiment
```bash
python -m experiments.defingerprint.run_defingerprint \
    --config experiments/defingerprint/config/experiment_config.yaml
```

### 2. Analyze Results
```bash
python -m experiments.defingerprint.analyze_results \
    --result_dir ./result/defingerprint_experiments/group_1
```

### 3. Visualize
```bash
python -m experiments.defingerprint.visualize \
    --result_dir ./result/defingerprint_experiments/ \
    --output ./result/defingerprint_experiments/plots/
```

## Prerequisites

Before running the experiment, ensure you have:

1. **Stage 1 Model** (optimal quality, no watermark):
   - Path: `./result/simpleunet_cifar10_stage1/model_final.pth`
   - Generatedby running: `bash ./script/simpleunet_cifar10_stage1.sh`

2. **Stage 2 Model** (with watermark and fingerprint):
   - Path: `./result/simpleunet_cifar10_stage2/model_final.pth`
   - Generated by running: `bash ./script/simpleunet_cifar10_stage2.sh`

3. **Environment Setup**:
   ```bash
   conda activate fedtracker  # or your environment name
   pip install torch torchvision diffusers transformers
   ```

## Experiment Groups

| Group | Description | Epochs | Expected Outcome |
|-------|-------------|--------|-------------------|
| **Group1** | Baseline (no fine-tuning) | 0 | Establish upper bound for fingerprint accuracy |
| **Group 2** | Light fine-tuning | 1 | Minimal quality loss, partial fingerprint removal |
| **Group 3** | Moderate fine-tuning | 3 | Noticeable quality impact, significant fingerprint removal |
| **Group 4** | Heavy fine-tuning | 5 | Quality degradation, substantial fingerprint removal |
| **Group 5** | Aggressive fine-tuning | 10 | Possible quality restoration, near-complete fingerprint removal |

## Configuration

All experiment parameters are defined in `config/experiment_config.yaml`:

### Model Parameters
```yaml
model:
  type: SimpleUNet
  dataset: cifar10
  num_classes: 10
  image_size: 32
```

### Fine-tuning Parameters
```yaml
finetuning:
  epochs: [1, 3, 5, 10]      # List of epoch counts to test
  learning_rate: 1.0e-4       # Fine-tuning learning rate
  local_ep: 5                # Local epochs per round
  local_bs: 64              # Batch size
```

### Fingerprint Parameters
```yaml
fingerprint:
  lfp_length: 128                    # Fingerprint length
  embed_layer_names: "mid_block.attention.proj"
  num_clients: 50                    # Number of clients
  epsilon: 0.5                       # Embedding strength
```

### Evaluation Metrics
```yaml
evaluation:
  fid_samples: 5000                  # Samples for FID calculation
  fingerprint_threshold: 0.85        # FSS score threshold
  seed: 42                           # Random seed
```

## Metrics Explained

### FID Scores
- **Normal Class FID**: Fréchet Inception Distance for normal CIFAR-10 classes
- **Trigger Class FID**: FID for trigger class generated samples
- Lower FID = Better generation quality

### Fingerprint Recognition
- **FSS (Fingerprint Similarity Score)**: Cosine similarity between extracted and original fingerprints
- **Accuracy**: Percentage of clients correctly identified
- **FSS Threshold**: 0.85 (configurable in config)

### Trade-off Analysis
- **Quality Loss**: `FID_finetuned - FID_baseline`
- **Fingerprint Removal**: `FSS_baseline - FSS_finetuned`
- **Effectiveness Ratio**: Fingerprint removal / Quality loss

## Analysis

The `analyze_results.py` script provides:

1. **Cross-group comparison** of fid metrics
2. **Fingerprint extraction accuracy** per epoch
3. **Client identification success rate**
4. **Statistical analysis** (mean, std, confidence intervals)

```bash
# Analyze individual group
python -m experiments.defingerprint.analyze_results\
    --result_dir ./result/defingerprint_experiments/group_2

# Analyze all groups
python -m experiments.defingerprint.analyze_results \
    --result_dir ./result/defingerprint_experiments/ \
    --compare_all
```

2. Use deterministic operations where possible
3. Save all intermediate results
4. Document GPU and PyTorch versions

## Troubleshooting

### CUDA Out of Memory
Reduce `fid_samples` in config or use smaller batch size:
```yaml
evaluation:
  fid_samples: 1000  # Reduce from 5000
```

### Slow FID Evaluation
Enable multi-GPU or reduce samples:
```bash
CUDA_VISIBLE_DEVICES=0,1 python -m experiments.defingerprint.run_defingerprint ...
```

### Missing Dependencies
```bash
pip install torch torchvision diffusers transformers tqdm pyyaml
```
